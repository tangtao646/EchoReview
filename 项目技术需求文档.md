# EchoReview 技术需求文档 (PRD)

## 1. 项目愿景
EchoReview 旨在成为面试者的“第二大脑”，通过稳定可靠的后台录音、精准的语音转文字（STT）以及智能化的 AI 总结，帮助用户在面试后快速进行归纳复盘，提升面试通过率。

---

## 2. 核心功能需求

### 2.1 长时间后台录音
*   **息屏录音**：支持在应用进入后台或屏幕熄灭时持续录音，确保面试全过程不间断。
*   **状态感知**：实时显示录音时长，提供暂停/继续功能。

### 2.2 语音转文字 (STT) 集成
*   **多模型扩展能力**：
    *   **国际**：OpenAI Whisper API, Google Speech-to-Text。
    *   **国内**：**阿里通义 (DashScope Paraformer)**, 科大讯飞, 腾讯云。
*   **处理模式**：支持录音完成后整体上传，以及分片流式上传。

### 2.3 AI 总结与分析
*   **大模型对接**：支持 DeepSeek (高性价比推荐), 通义千问 (Qwen), GPT-4o 等主流云端模型。
*   **本地推理 (未来扩展)**：预留对本地轻量化模型 (如 Qwen-1.5B-Chat, Gemma) 的支持，确保极端隐私场景下的离线复盘。

### 2.4 本地库管理
*   **持久化存储**：存储录音、全量转写文本、AI 总结、面试日期及公司标签。

---

## 3. 技术架构设计

### 3.1 移动端技术栈
*   **UI 框架**：Jetpack Compose。
*   **异步流**：Kotlin Coroutines + Flow (用于实时处理音频数据流)。
*   **音频采集**：**AudioRecord API** (采集原始 PCM 数据，满足 AI 模型对采样率和位深度的精确要求)。
*   **音频编码**：**MediaCodec** (将 PCM 实时转码为 AAC/Opus 以降低本地存储占用)。
*   **数据持久化**：Room Database (文本 with metadata) + DataStore (配置项)。
*   **本地 AI 推理层 (预留)**：预留对 **MediaPipe LLM Inference SDK** 或 **ONNX Runtime** 的集成接口。

### 3.2 关键技术实现路径
1.  **流式录音架构 (Streaming Audio Architecture)**：
    *   使用 `AudioRecord` 开启 16kHz/16bit/Mono 采样任务。
    *   通过 `Kotlin Flow` 构建生产者-消费者模型：
        *   **分流 A (存储)**：送入 `MediaCodec` 进行 AAC 压缩，保存为 `.m4a` 文件。
        *   **分流 B (AI)**：(可选) 实时切片发送至云端 STT 流式接口（如 Paraformer Realtime）。
2.  **前台服务录音**：使用 `Foreground Service` 配合持久通知栏，并声明 `microphone` 服务类型，确保息屏不中断。
3.  **API 策略模式 (Extensible API Strategy)**：
    *   定义统一的 `STTEngine` 和 `LLMProvider` 接口，支持通义千问、DeepSeek 等。
    *   支持用户自定义 API Key，并通过工厂模式动态实例化。

## 4. 数据结构预览 (Room)
*   **Table: `InterviewSession`**
    *   `id`: UUID
    *   `companyName`: String
    *   `timestamp`: Long
    *   `audioFilePath`: String
    *   `transcription`: String
    *   `aiSummary`: String
    *   `aiModelUsed`: String (记录本次使用的模型名称，便于对比结果)
    *   `isLocalInference`: Boolean (标记是否为本地模型推理)
    *   `tags`: List<String>

---

## 5. 风险与规避方案
*   **API 密钥安全**：本地加密存储 API Key，建议中长期采用 Backend-for-Frontend (BFF) 模式进行代理请求。
*   **离线可用性**：即便没有网络，也优先确保录音保存，待网络恢复后再触发云端任务。

---

## 6. 开发路线图 (MVP)
*   **Phase 1**：实现前台服务录音与本地 Session 管理。
*   **Phase 2**：通过策略模式集成 **通义千问 (Paraformer + Qwen)** 或 DeepSeek。
*   **Phase 3**：优化 UI/UX，添加多模型切换和面试标签系统。
*   **Phase 4**：探索本地端侧大模型实现（如使用 Qwen-1.8B 等轻量模型进行本地总结）。
