# EchoReview 架构规约与目录结构 (Clean Architecture & MAD)

## 1. 架构原则
本项目遵循 **Clean Architecture (整洁架构)**，核心目标是**业务逻辑与技术细节解耦**。

*   **Domain 层 (领域层)**：纯 Kotlin 模块，包含业务实体 (Entities)、用例 (UseCases) 和存储库接口 (Repository Interfaces)。它不依赖 Android 框架（无 Context）。
*   **Data 层 (数据层)**：负责数据持久化（Room）、网络请求（Retrofit/DashScope）和硬件交互（AudioRecord）。它是 Domain 层接口的实现者。
*   **UI 层 (表示层)**：采用 **MVI (Model-View-Intent)** 或 **MVVM** 架构，使用 Jetpack Compose 实现，严格遵循单向数据流 (UDF)。
*   **DI 层 (依赖注入)**：使用 **Hilt** 全局管理依赖，确保各层解耦。

## 2. 目录结构规范

```text
app/src/main/java/com/dream/echoreview/
├── common/                # 通用工具类、常量、扩展函数
├── di/                    # Hilt Modules (AppModule, NetworkModule, AudioModule)
├── domain/                # 领域层 (最内层)
│   ├── model/             # 纯 Kotlin 业务对象 (InterviewSession)
│   ├── repository/        # 仓库接口协议 (IInterviewRepository, IAudioRecorder)
│   └── usecase/           # 业务逻辑单元 (StartRecordingUseCase, GetAIResultUseCase)
├── data/                  # 数据层 (中层)
│   ├── local/             # Room Database, DAO, Entity (DB 映射对象)
│   ├── remote/            # API Service, DTO (网络映射对象), DashScope 实现
│   ├── repository/        # Repository 接口的具体实现
│   └── audio/             # AudioRecord 具体实现 (PCM 采集与转码逻辑)
└── ui/                    # 表示层 (最外层)
    ├── theme/             # Compose 主题与样式
    ├── component/         # 通用 Compose 组件 (按钮、列表项等)
    ├── screen/            # 各功能页面目录 (Home, Recording, Detail)
    │   ├── HomeViewModel.kt
    │   └── HomeScreen.kt
    └── MainActivity.kt
```

## 3. 核心功能接口约定 (Interface Contracts)

### 3.1 录音采集协议 (`domain/repository/IAudioRecorder.kt`)
```kotlin
interface IAudioRecorder {
    fun start(outputFile: File)
    fun stop()
    fun pause()
    fun resume()
    val audioFlow: Flow<ByteArray>   // 暴露实时 PCM 数据流供 STT 使用
    val stateFlow: StateFlow<RecordingState>
}
```

### 3.2 AI 策略协议 (`domain/repository/IAIService.kt`)
```kotlin
interface ISTTEngine {
    suspend fun transcribe(audioFile: File): Result<String>
    suspend fun transcribeStream(pcmChunk: ByteArray): Flow<String>
}

interface ILLMProvider {
    suspend fun generateSummary(transcript: String): Result<String>
}
```

## 5. MAD 技术栈强制约束
1.  **依赖注入**：所有 Repository 和 Service 必须通过 Hilt 注入。
2.  **异步处理**：禁止使用 `Thread`，统一使用 `Coroutines`。UI 层必须使用 `viewModelScope`。
3.  **数据交互**：Data 层对象 (Entity/DTO) 传往 Domain 层前必须进行 `toDomain()` 映射转换。
4.  **UI 状态管理**：使用 `StateFlow<UiState>`，状态必须是不可变对象 (Immutable)。
